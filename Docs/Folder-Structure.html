📁 Directory Breakdown

Frontend Structure (Next.js App Router)

Organized by route groups and concerns. Authentication and dashboard are separate route groups for cleaner middleware handling.

app/ - Application Pages and API Routes

  • (auth)/ - Public authentication pages (login, register)
  • (dashboard)/ - Protected recruiter/admin interface
  • api/ - REST API endpoints for all operations

lib/ - Shared Utilities and Business Logic

  • db/ - Database client, schema definitions, and query helpers
  • redis/ - Redis singleton for caching and rate limiting
  • queue/ - BullMQ job queue setup and worker process
  • cron/ - Scheduled jobs (reminders, digests)
  • email/ - Email templates and transactional email client
  • validations/ - Zod schemas for request validation
  • utils/ - Helper functions (rate limiting, caching)

components/ - Reusable React Components

  • kanban/ - Kanban board components
  • forms/ - Form components and handlers
  • ui/ - Base UI components (buttons, cards, modals)

Root Files

  • middleware.ts - Route protection and auth guards
  • worker.ts - Entry point for background worker process
  • drizzle.config.ts - Database migration configuration

🏛️ Architecture Patterns

API Conventions

  • RESTful endpoints with standard HTTP methods
  • Validation on all POST/PATCH endpoints using Zod
  • Atomic transactions for multi-step operations (e.g., stage changes)

Background Jobs

  • Long-running tasks (resume parsing, emails) handled asynchronously
  • BullMQ manages retry logic and dead-letter queues
  • Independent worker process for horizontal scaling

Database Layer

  • Drizzle ORM for type-safe queries
  • Migrations tracked in drizzle/ folder
  • All tables use UUID primary keys for scalability

🚀 Running the Application

Frontend (Next.js on port 3000)

cd frontend
npm run dev

Backend (Node.js API on port 5000)

cd backend
npm run dev

Background Worker (Separate Process)

cd frontend
npm run worker

Database Migrations

cd frontend
npm run db:push
